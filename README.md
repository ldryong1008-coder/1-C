# Mini NPU Simulator

이 프로젝트는 Neural Processing Unit(NPU)의 핵심 연산인 MAC(Multiply-Accumulate) 연산을 Python으로 구현한 시뮬레이터입니다. 입력된 2차원 패턴이 두 가지 필터(십자가, X) 중 어느 것과 더 유사한지를 판별합니다.

## 실행 방법

- **요구 사항**: Python 3.8 이상 (외부 라이브러리 필요 없음)
- **실행**: 터미널에서 다음 명령어를 입력합니다.
  ```bash
  python main.py
  ```
- **모드 선택**:
  1. `1. 사용자 입력 (3x3)`: 3x3 크기의 필터 A, B와 패턴을 직접 터미널에 입력하여 결과를 확인합니다. 각 줄은 띄어쓰기로 구분된 숫자 3개로 입력해야 합니다.
  2. `2. data.json 분석`: 실행 폴더에 위치한 `data.json` 파일을 읽어 일괄 판정을 수행합니다. 5x5, 13x13, 25x25 크기의 패턴 및 필터 검증과 성능 측정이 포함되어 있습니다.

## 구현 요약

- **라벨 정규화 (Label Normalization)**: JSON 스키마에서 예상값(`expected`)과 필터 키(`keys`)는 대소문자나 표기 방식이 다양할 수 있습니다(`+`, `x`, `cross`, `Cross`, `X`). 이를 내부적으로 비교하기 쉽게 `Cross` 와 `X` 두 가지 표준 라벨로 통일(정규화)하여 처리합니다.
- **MAC 연산**: 입력 패턴과 필터를 겹쳐서 같은 위치의 숫자끼리 곱한 결과를 누적 합산합니다. 벡터화 라이브러리(NumPy 등)를 사용하지 않고 중첩 반복문(for-loop)을 사용하여 수동으로 구현하였습니다.
- **동점 처리 정책 (Epsilon Equality)**: 부동소수점 오차로 인해 발생할 수 있는 미세한 값 차이를 무시하기 위해 허용오차(`epsilon = 1e-9`) 기반 비교를 수행합니다. 두 점수의 차이가 `1e-9` 미만일 경우 동점으로 판정하며, 최종 결과를 `UNDECIDED`로 반환합니다.

## 결과 리포트

### FAIL 케이스 원인 분석
`data.json` 테스트 실행 시 다음과 같은 두 가지 유형의 FAIL이 발생할 수 있도록 설계되었으며, 이를 분석한 결과는 다음과 같습니다:

1. **데이터/스키마 문제 (예: `size_5_mismatch`)**:
   입력 패턴의 크기가 예상된 필터의 크기(NxN)와 맞지 않는 경우입니다. 이 경우 연산을 진행하면 인덱스 오류가 발생할 수 있으므로 시뮬레이터는 이를 감지하여 프로그램 중단 없이 해당 케이스를 즉시 `FAIL` 처리하고 에러 사유(패턴 크기 불일치)를 반환합니다.
2. **수치 비교 문제 / 로직 문제 (예: `size_13_1`)**:
   두 필터(Cross, X)와의 MAC 점수가 완벽하게 동일하거나, 부동소수점 오차 범위 내에서 동일하다고 판정된 경우입니다. 동점 규칙(Epsilon 정책)에 따라 `UNDECIDED`로 판정되지만, `data.json`의 기대값(`expected`)은 특정한 필터를 요구하기 때문에 불일치로 `FAIL` 처리됩니다. 이는 알고리즘이 정상 동작하여 모호한 판단을 유보한 올바른 결과입니다.

만약 모든 케이스가 PASS(`0 FAIL`) 했다면, 이는 제공된 데이터가 올바른 스키마를 갖추고 있고 라벨 정규화 및 부동소수점 오차 허용 로직이 성공적으로 작동하여 비교 오류가 발생하지 않았음을 의미합니다.

### 성능 표 해석 및 시간 복잡도 O(N²) 분석
각 패턴 크기(N)가 커짐에 따라 측정된 평균 MAC 연산 시간은 다음과 같은 추세를 보입니다. (측정 환경에 따라 절대값은 다를 수 있습니다.)

| 크기 (NxN) | 연산 횟수 (N²) | 평균 시간(ms) |
|------------|----------------|---------------|
| 3×3        | 9              | 0.004         |
| 5×5        | 25             | 0.009         |
| 13×13      | 169            | 0.051         |
| 25×25      | 625            | 0.181         |

**시간 복잡도 O(N²) 근거**:
표에서 볼 수 있듯이, 5x5(연산 25회) 대비 25x25(연산 625회)는 `N`이 5배 증가했지만, 총 연산 횟수(`N²`)는 25배 증가했습니다. 평균 시간 역시 약 0.009ms에서 약 0.181ms로 약 20배가량 증가하여, **총 연산 횟수의 증가율과 평균 실행 시간의 증가율이 거의 비례**함을 알 수 있습니다.
이는 MAC 연산이 중첩된 두 개의 `for` 반복문(가로 길이에 대해 N번, 세로 길이에 대해 N번)을 통해 수행되므로, 처리해야 하는 전체 원소의 개수인 `N * N = N²`에 정비례하는 실행 시간을 요구하기 때문입니다. 따라서 이 알고리즘의 시간 복잡도는 O(N²)으로 설명할 수 있습니다.
